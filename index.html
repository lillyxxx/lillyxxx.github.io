如何利用GitHub Pages搭建自己的博客（基于Hexo）

Posted on 2019-02-23 | Comments: 0 | Views:
Symbols count in article: 3.6k | Reading time ≈ 3 mins.

本文基于Mac OS操作系统

准备工作

  1. 首先需要安装的是Git，node.js

    • 安装git可用 $ brew install git

    • 安装 node.js 推荐使用 nvm

      nvm 是安装node.js的最佳方式

      • 推荐使用curl安装！！！（brew安装会出现重启terminal nvm命令失效）

        — 如果已经使用brew安装过，可以先卸载nvm $ brew uninstall nvm 再执行后续操作

      • 重启terminal，输入 $ nvm --version 查看nvm是否安装成功

        — 如果出现 nvm: command not found ，可能是并没有 ~/.bash_profile 文件存在，可以按照以下方式尝试解决：

        — 然后重启terminal

      • 安装Node.js （默认stable版本）

      • $ nvm install stable

    • 至此，node.js安装成功，可用$ nvm ls查看已安装的node 版本。

      • 可能有人会出现在安装nvm之前已经通过.pkg安装过了node.js，则安装的node版本并不会出现在nvm里，如果想卸载已安装的node从nvm中安装新的node，可参考如下方法：

      • 删除nvm文件

      • 同时可能还需要

      • 最后通过Homebrew检查

      • 按照需求执行clean或prune

  2. 安装 Hexo

    $ npm install -g hexo-cli

    可参考： Hexo 中文站

  3. 如果是使用brew安装，则关闭一次terminal可能会出现发现nvm和hexo均不能运行，尝试$ export NVM_DIR="$HOME/.nvm" 和 $ . $(brew --prefix nvm)/nvm.sh 可以解决，也可以把这两句加在 ~/.bash_profile文件里，但这样会导致每次重启terminal花的时间较长

在本地上配置blog

  1. 初始化Hexo

    • 首先用hexo初始化blog

      这里推荐 为GitHubusername.github.io。 因为如果没有购买域名，那你的blog所在的地址就是GitHubusername.github.io

    • 此时hexo已经初始化完毕可以进行一些预览，Hexo附赠一篇hello world文章

      在浏览器输入 http://localhost:4000/ 就可以看见网页和模板了

      • 如果对这个主题满意，可以跳过修改主题的步骤
      • 在本地预览模式下可以实时更新你的blog，即如果需要修改文章，可以不用退出server直接修改，文章修改好保存之后，直接刷新页面就可以看到更改后的情况。

    • _config.yml 文件里包含的就是你的blog网站的配置信息，可以对其进行更改（可参考官方文档）

  2. 更换Hexo的主题

    • Themes模版可以在Hexo官网上找到，点击这里
    • 推荐主题NexT，有非常详细的说明和教程
    • 本文以Cactus为例

    更改之后需要重新运行 $ hexo g 和 $ hexo s。

  3. 新建一篇自己的文章（Hexo官方教程）

    • 在博客中创建新的文章主要依靠以下这条指令

      如果不指定 layout 那么就按照在文件_config.yml 里的default_layout 的参数值默认

      其中Hexo提供三种不同的layout，包括 post ， page 和 draft。

      • Post

        存放在GitHubusername.github.io/source/_posts/ 文件夹下，即一般的博文，采用.md格式

      • Page

        存放在GitHubusername.github.io/source/ 文件夹下，相当于”栏目“，比如对应Cactus主题下，about就是一个page。新建了page之后会在source文件夹下新建同名文件夹，里面包含一个基本的index.md文件

      • Draft

        存放在GitHubusername.github.io/source/_draft/ 文件夹下，顾名思义草稿，写好的draft可以通过 $ hexo publish [layout] <title> 来移动到posts里

    • 也可以在各种layout对应的文件夹下直接创建文章，这样也可以生成对应的post/draft，注意文件一定是.md 格式的。

    • 删除文章可以直接删除本地文件，再重新generate静态文件

将blog配置到GitHub上

  • 首先安装Hexo关于GitHub的支持插件，这样可以让以后的自动部署发布方便很多。关于这个插件

  • 在自己的GitHub上创建自己blog的repository，起名为username.github.io

    • 注意这里要选择public repository，针对private repository的GitHub pages目前只支持GitHub Pro， GitHub Team， GitHub Enterprise Cloud， 以及 GitHub Enterprise Server。

  • 修改_config.yml文件

  • 执行 $ hexo clean && hexo deploy 清除站点文件并上传新的文件

  • 然后在GitHub上找到这个repository，在设置里面将 Branch>>Default Branch 选为在 _config.yml文件里设置的branch即可，如果没有设置默认master

好啦！现在访问 https://username.github.io 就可以看到你的个人博客了
